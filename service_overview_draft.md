# **1. 요약**

### **Pain Point**

---

- 문서가 여러 페이지에 흩어져 있어 **용어/정의/설명 일괄 수정 어려움**
- 문서 간 관계를 한눈에 보기 어려움 → **구조 파악 불가, 탐색 난이도 ↑**
- 초보자/전문가 모두에게 Docs가 **동일한 흐름**으로 제공
- Docs 내 효율적인 정보 탐색 한계 → **AI로 문맥 기반 탐색 제공**

### **핵심 차별점**

---

1. **용어집 기반 일괄 수정 (Glossary ↔ 전체 문서 자동 연동)**
    - 단어 정의 변경 시 연결된 모든 페이지 자동 업데이트
    - 문서 간 consistency 확보
2. **지식그래프 뷰에서 전체 문서 구조 시각화**
    - `페이지 = 노드`  `문서 연결 = 엣지`
    - 온톨로지 Entity Class별 구분
        - 태그 그룹(#Beginner 등) 구분
    - 수정 영향 범위 하이라이팅
3. **문서 Layer 구조 (키워드·요약·정문) 자동 생성**
    - Layer를 기반으로 문서 추천 정확도 강화
        - Layer 1: AI가 추출한 핵심 개념 (Concept/Entity 매핑)
        - Layer 2: 요약본
        - Layer 3: 원문
4. **사용자 맞춤형 문서 추천 UX**
    - 사용자 맞춤형 온보딩
    - 현재 읽는 페이지 + 연결 노드 + 온톨로지 기반 추천
    - Assistant: 현재 문서와 연결된 온톨로지 데이터를 컨텍스트로 자동 주입
    - 북마크 문서는 그래프에서 시각적으로 표시

# **2. 상세 구조화 (서비스 전체 Blueprint)**

### **관리자(Admin) 기능**

---

**1) 문서 레이어화 및 AI 자동 생성**

- Layer 1: **핵심 개념 키워드** 추출 → 온톨로지 Entity로 매핑
- Layer 2: 요약문 생성 → 초보자/검색용
- Layer 3: 원문 저장

**2) Glossary 기반 자동 일괄 수정**

- Glossary는 단순 치환이 아니라 **온톨로지 기반 개념 정의 Entity**
- Glossary에서 용어/개념 정의 등록 시, 해당 개념을 사용하는 모든 문서에 자동 반영
- Glossary 수정 시 영향받는 문서를 그래프에서 하이라이트 표시

예시: “API → Application Programming Interface”로 변경 → 개념 Entity(API)를 참조하는 모든 문서가 자동 업데이트

**3) 지식 그래프(관리자용)**

**목적: 전체 문서 구조 및 개념 기반 영향도 분석**

- 노드에 담기는 정보(예시):
    - 문서 제목
    - 태그 또는 개념 Class
    - Layer1 개념 구조
    - 연결된 문서 수
- 엣지 정보(예시):
    - 문서 간 직접 링크
    - 공통 개념 기반 유사도
    - Entity Graph로부터 파생된 의미 기반 연결
    - 가중치(weight)
- 관리자 기능:
    - 노드 클릭 → 상세 정보 표시
    - Entity 필터링
    - 수정 영향도 시각화
    - 새 문서 추가 시 연결 위치 추천

**4) 톤앤매너 관리 기능**

AI가 자동으로 체크 or 관리자 설정 기능

- 문체 통일(말투/높임말 등)
- 스타일 규칙 적용
- 어투 검사 및 수정안 제시

**5) 관리용 디렉토리/카테고리 UI**

Obsidian의 sidebar 구조 참고 → 관리자 입장의 문서 관리 흐름 개선 

- Obsidian 스타일 Sidebar
- 문서, 개념 카테고리, Glossary 구조 모두 관리 가능
- 문서 + 개념 + 그래프가 통합된 관리 구조

### **사용자(User) 기능 구조**

---

**1) 온보딩 & 문서 추천 UX**

사용자 최초 방문 

- 태그/개념 기반 문서 추천
- Layer2 요약 기반 빠른 탐색
- 유사도 분석을 통한 Learning Path 제공

**2) 문서 뷰 + AI 대화**

문서를 읽는 동안 AI가 아래 정보를 컨텍스트로 활용

- 현재 문서의 핵심 개념(Entity)
- 연결된 문서(Document graph 기반)
- 개념 계층(ontology graph)
- 요약본
- 같은 태그/개념 Class 문서

**3) 문서 간 이동 (탐색 추천)**

- 온톨로지 기반 개념 유사도
- 태그/초보자 우선순위
- Graph weight 기반 다음 문서 추천

**4) 사용자용 그래프뷰(보조)**

- 내가 본 문서 강조
- 북마크 문서 강조
- 관련 문서 그룹 펼쳐보기
- 초보자용 필터 제공 : 온톨로지 기반 관계도로 정교한 탐색

**확장) 플랫폼화** 

- 여러 서비스의 Docs를 통합하여 비교/탐색 가능
- 장기적으로 B2B SaaS 형태로 제공 가능
- 문서 구조 자동 분석, 추천 엔진 기반 API로 확장 가능

---

- **GraphDB**
    - Document 노드 사용 (메타 정보 + 엣지)
    - Concept 노드, Tag 노드, Version 노드 등
    - Document 간 parent-child / group 관계, Document ↔ Concept 관계 저장
- **RDB**
    - 각 Document의 본문, 이미지, 버전별 content 등 **실제 컨텐츠** 저장

→ 둘이 **document_id**로 연결되는 구조

- Document 계층, Document–Concept, Document–Document 관계는 GraphDB의 Document Node 및 엣지로 관리
- 문서의 실제 컨텐츠(본문/예제/이미지) RDB에 저장
- Document Node는 doc_id 기준으로 RDB의 문서 row와 연결
- Graph는 구조 탐색/추천/그래프뷰, RDB는 렌더링 및 버전 관리를 담당
